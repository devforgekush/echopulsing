## 📞 Support & Troubleshooting

### Common Questions

**Q: Will this affect performance?**  
A: No. Status check uses cached data. Performance impact negligible.

**Q: What if the status check fails?**  
A: Returns empty string. Message displays normally. Zero impact.

**Q: Can we customize the emoji?**  
A: Yes. Edit the f-string in `_get_assistant_status()` method.

**Q: Do we need to restart anything?**  
A: Yes, restart the bot after deploying the new file.

**Q: How do we roll back if there's an issue?**  
A: Restore previous `player_ui.py` and restart. Very easy.

### Troubleshooting

**Status not showing?**
1. Check bot is in voice chat
2. Verify assistant_label is set
3. Check error logs
4. See UI_IMPROVEMENT_ASSISTANT_STATUS.md "Troubleshooting"

**Status showing incorrectly?**
1. Verify is_in_chat() returns correct value
2. Check HTML formatting is valid
3. See CODE_REFERENCE.md "Error Handling"

**Performance degradation?**
1. Check for other running tasks
2. Monitor status check duration
3. See CODE_REFERENCE.md "Performance Considerations"

**Display cut off on mobile?**
1. Check message length
2. Verify emoji display
3. See UI_VISUAL_SHOWCASE.md "Mobile View Example"
